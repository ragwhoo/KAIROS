Gradient Directions

Where gradient elevates the experience vs. where flat is better.

Three Gradient Approaches

A. Accent-Only Gradient

Only gradient on primary accent elements (buttons, active states, progress bars). Everything else stays flat. Most restrained Apple-like approach.

B. Full Gradient Identity

Gradient on accent + backgrounds + gamification elements. Page BG has subtle gradient. Cards use gradient accents. XP & levels are gradient-driven.

C. Glass + Gradient

Gradients paired with glassmorphism. Cards have subtle frosted-glass backgrounds with gradient borders. Feels modern but risks feeling heavy.

Approach A: Accent-Only Gradient (Recommended)

Gradient only on the accent. Everything else stays flat. This is what Apple would do — restraint makes the gradient feel special.

Gradient Guidelines

Gradient: #0071e3 → #6951e3 (30° hue shift, same saturation)

Use on

  • Primary buttons
  • XP / level badges
  • Progress bars
  • Active nav item

Avoid on

  • Body text (hard to read)
  • Small UI elements
  • Cards / containers
  • Error states (clarity matters)
